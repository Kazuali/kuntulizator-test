Кунтулизатор v32 — плей-офф 1/16
=================================

Что изменено в v32:

1. Добавлен этап 1/16 финала.
   - Ключ этапа: r4
   - Название листа Google Sheets: 1-16
   - CSV gid листа: 256656047
   - Дедлайн сдачи этапа: 2026-06-29T20:00:00+03:00
   - Первый матч ЮАР — Канада остаётся в расчётах, но не показывается в форме сдачи через сайт.
   - Сдача через сайт начинается со второго матча этапа.

2. Новая логика плей-офф.
   - Прогноз = счёт + команда, которая проходит дальше.
   - Если счёт не ничейный, сайт сам выбирает победителя по счёту.
   - Если счёт ничейный, участник обязан выбрать команду, которая проходит дальше.

3. Новый банк плей-офф.
   - Исход / проход: 1600 очков при 16 участниках.
   - Точный счёт + правильный проход: 1600 очков при 16 участниках.
   - Итого: 3200 очков за матч.
   - Групповой этап не пересчитывается по новым правилам: там остаётся 100 + 50 с активного участника.
   - В 1/16 стоит 100 + 100 с активного участника.

4. Правила начисления в плей-офф.
   Пример: факт 1-1 Бразилия.
   - 1-1 Япония: 0 очков.
   - 1-1 Бразилия: очки за проход + очки за точный счёт.
   - 2-2 Бразилия: только очки за проход.

5. Формат таблицы.
   В строке РЕЗУЛЬТАТ можно писать:
   - 2-1
   - 0-1
   - 1-1 Бразилия
   - 2-2 Япония

   В прогнозах участников такой же формат.
   Для неничьей можно писать только счёт, победитель будет понятен по счёту.
   Для ничьей нужно обязательно указать команду.

6. Файлы версии:
   - index.html
   - data-v32.js
   - app-v32.js
   - styles-v32.css
   - google_apps_script_kuntulizator_v7.txt

Как заливать на тест:

1. В репозиторий kuntulizator-test загрузить:
   - index.html
   - data-v32.js
   - app-v32.js
   - styles-v32.css
   - hero-messi.jpg, если его нет или он изменился

2. НЕ загружать на GitHub:
   - google_apps_script_kuntulizator_v7.txt

3. Код из google_apps_script_kuntulizator_v7.txt вставить в Google Sheets:
   Расширения → Apps Script.

4. После обновления Apps Script проверить, что Web App URL в data-v32.js остался рабочим.
   Если Apps Script даст новую ссылку деплоя, заменить submissionWebAppUrl в data-v32.js.

5. Проверка на тестовом сайте:
   - открыть https://kazuali.github.io/kuntulizator-test/
   - вкладка «Сдать прогноз» должна открываться на 1/16 финала;
   - в форме первым матчем должен быть Бразилия — Япония, а ЮАР — Канада не должен быть доступен для сдачи;
   - при 2-1 автоматически подсвечивается первая команда;
   - при 1-2 автоматически подсвечивается вторая команда;
   - при 1-1 можно выбрать любую команду;
   - прогноз 1-1 без выбранной команды не должен отправляться;
   - после внесения результата 1-1 Бразилия проверить начисления:
     1-1 Япония = 0,
     1-1 Бразилия = исход + точный,
     2-2 Бразилия = только исход.
